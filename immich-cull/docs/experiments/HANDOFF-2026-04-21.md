# Handoff: 2026-04-21 — auto-cull is shipped

Supersedes `HANDOFF-2026-04-20.md` (still worth reading for context on *why* we changed lenses).

---

## What's shipped

The production pipeline now does, in order, per batch:

1. Session batcher splits the library into session-coherent batches ≤30 photos.
2. LLM ranks each batch (`gemini-3.1-flash-lite-preview` + 92-line `SYSTEM_PROMPT`).
3. **Face-coverage post-check** promotes culls→keeps when a named person would otherwise be dropped from the batch. Named-only, deterministic, ~free.
4. `classifyBatchForAutoCull` tiers each photo `auto-cull-high` / `auto-cull` / `review`.
5. User approves (single-model `auto-approve`, multi-model `approve-confident`) or reviews manually.

Full details: `docs/PIPELINE.md`.

Disable face-coverage with `DISABLE_FACE_COVERAGE=1` if it ever misbehaves.

---

## Why this is ship-ready

Measured on 80 graded batches (`data/experiments/2026-04-2*-batch-*-grades.json`):

| pipeline | acceptable-rate | sev-2 | sev-3/4 |
|---|---:|---:|---:|
| v1 LLM alone (baseline) | 96.2% | 3 | 0 |
| v1 + all-cluster face-cover | 95.0% | 4 | 0 |
| **v1 + named-only face-cover** | **97.5%** | **2** | **0** |

- Zero severity-3/4 across 80 batches. Rule-of-three upper bound ≤3.7%.
- The one "face-coverage exists to fix this" case — the breakfast/Thomas regret — is fixed.
- Named-only strictly dominates both alternatives: added no new regressions, removed one.

Codex independent review (full transcript in session log) verdict: *"Ship v1_prod as the current baseline with a targeted guardrail."* Done.

## What changed vs. the 2026-04-20 handoff

| Idea | Status | Why |
|---|---|---|
| Pure-burst cull only | **Abandoned** | The big insight — subgroup-lens grading is the wrong metric. Batch-lens grading ~doubles the "acceptable" count. Pure-burst detector was designed for a problem that doesn't exist at batch scale. |
| v3 adaptive prompt (subgroup) | **Archived** | Real F1 lift at subgroup level, but subgroup F1 was the wrong target. Kept in archive for reference. |
| New batch prompts (adaptive / priorities / v1_style / min) | **Dropped** | All within 2-4pp of v1_prod's 89.7% acceptable-rate on small samples. Not worth hunting. `min` was a clear loss, confirmed. |
| Face-coverage as tie-breaker (§4.4 old handoff) | **Shipped, reframed** | Not a tie-breaker — a post-check. Not in prompt — in code. Named-only, not all clusters. |
| Two-stage local+cloud pipeline | **Still not needed** | Cloud gemini-3.1-flash-lite is ~$0.0001/batch, no latency issue. Local models slower AND weaker at batch scale. |
| Confidence self-reporting | **Not pursued** | Codex argued LLM-reported confidence is weakly calibrated. Deterministic guardrails (face-cover) target the actual failure mode better. Keep as future option if tail reappears. |

## Metrics — what we measure now and why

- **Acceptable-rate** (primary): % of user-graded picks with severity ≤ 1. Honest product metric for "would I trust this for auto-cull?".
- **Severity-2 count** (regret guardrail): the sad-but-survivable failures. Watch this go up under a change.
- **Severity-3/4 count** (catastrophic guardrail): painful misses. Currently 0. A single severity-3 would put a new variant on probation.
- **Retain-rate** (gaming-protection): keep-count / batch-size. Prevents a "keep everything" model from gaming acceptable-rate. `gemma4:e4b` at 98–100% retain rate is the cautionary example.

**F1** is explicitly deprecated as the primary metric. It treats near-duplicate substitutions as full misses (user is mostly indifferent). Useful only as a coarse filter — "is this variant in the right ballpark?".

## Known limitations (explicit, not sweep-under-rug)

1. **Screenshots are context-dependent**. Snapchat saves → user wants kept (social memory); sleep-app dev screenshots → user wants culled (ephemeral). The LLM can't reliably distinguish origin from pixels. Fix requires filename patterns / app-origin metadata, not prompt work.
2. **Old scanned photos** trigger "keep more than I would" mode (prompt is tuned for family photos). Rare enough to leave.
3. **Face-cover only protects named people**. Untagged people get no protection. Mitigation: tag more people in Immich over time.
4. **Unnamed face clusters ignored** by face-coverage. Including them regresses acceptable-rate by 1.2pp — the clustering is too noisy. Revisit if Immich ever exposes cluster confidence scores.

## What's worth doing next

Ordered by impact × effort.

### 1. Run current pipeline on the 72k backlog. Staged.

You have ~16k already burst-auto-culled and 3,843 manual decisions. That leaves ~52k photos across whatever batches haven't been ranked yet. Staged rollout:

1. Run the LLM on a wave (say 500 batches) — this populates the cache.
2. Let `getAutoCullSummary` produce tier lists (face-cover applies automatically).
3. User reviews the HIGH tier for a sample (~30 batches) to confirm the shipped numbers hold up on unseen data.
4. If confirmed, bulk-apply HIGH tier via `/api/batches/auto-approve` across the 500-batch wave.
5. Repeat waves until the backlog is consumed.

No code needed — the pipeline is ready. Just a sequencing/UX task.

### 2. Screenshot filename rules (30 min).

`auto_keep_patterns` / `auto_cull_patterns` matched against filename before the LLM sees the batch. From user notes:
- `Snapchat-*` → auto-keep (or at least "bias towards keep")
- Sleep-app / dev screenshots (pattern from filename) → auto-cull

Saves LLM work on easy cases and fixes the recurring pain point.

### 3. More graded batches (optional).

Current Wilson 95% CI on acceptable-rate is ~91–99%. If you want to claim ≥95% with high confidence you'd need ~150–200 more graded batches. Only worth doing if bulk rollout surfaces surprises.

### 4. (Speculative) Light confidence signal.

If the backlog rollout surfaces a new class of severity-2 failures that face-cover can't touch, revisit self-reported LLM confidence — but with a calibration plot, not blind trust. Don't do until there's a documented failure pattern.

## Operational notes

- Server: `npx tsx src/server.ts --immich-api --vertex --port 3737`. Face-coverage auto-enables when `IMMICH_URL`/`IMMICH_API_KEY` are set.
- Grader UI: `http://localhost:5173/#experiment?id=<experiment_id>`. `Shift+→` skips to next group with ungraded bundles.
- Grade inheritance works across experiment files by `(group, picks)` key. If you grade a pick in one file, it auto-applies to every other file where the same batch has the same keep set.
